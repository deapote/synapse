# 参考 —— 错误处理

## 通用响应格式

所有 API 返回统一的 JSON 结构：

### 成功响应

直接返回业务数据，无统一包装。

### 错误响应

```json
{
  "message": "错误描述",
  "traceId": "可选的追踪 ID"
}
```

## 错误码对照表

| HTTP 状态码 | 场景 | 示例 message |
|------------|------|-------------|
| 400 | 请求参数非法 | "分页参数非法" |
| 400 | 领域规则校验失败 | "文件名不能为空" |
| 400 | 文件大小超过限制 | "文件大小超过限制" |
| 400 | 不支持的文件类型 | "不支持的文件类型" |
| 401 | 未登录或 token 过期 | "未登录" |
| 403 | 权限不足 | "无权限访问" |
| 404 | 资源不存在 | "未找到知识库: xxx" |
| 404 | 文档不存在 | "未找到文档: xxx" |
| 409 | 业务冲突 | "此知识库已存在相同内容的文档" |
| 500 | 内部服务器错误 | "流式问答失败" |

## 常见错误场景

### 400 Bad Request

```json
{
  "message": "分页参数非法"
}
```

### 401 Unauthorized

```json
{
  "message": "未登录"
}
```

### 403 Forbidden

```json
{
  "message": "当前用户没有该权限"
}
```

### 404 Not Found

```json
{
  "message": "未找到知识库: kb-uuid"
}
```

### 409 Conflict

```json
{
  "message": "此知识库已存在相同内容的文档"
}
```

## 全局异常处理

所有异常由 `GlobalExceptionHandler` 统一处理：

- `DomainException` → 400 Bad Request
- 认证异常 → 401 Unauthorized
- 权限异常 → 403 Forbidden
- 其他异常 → 500 Internal Server Error

生产环境不暴露内部堆栈给前端。
