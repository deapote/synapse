# API 参考 —— 知识库

## POST /api/knowledge-bases

创建知识库。

### 请求

```bash
curl -X POST http://localhost:8080/api/knowledge-bases \
  -H "Content-Type: application/json" \
  -H "satoken: xxx" \
  -d '{"name":"产品手册","description":"公司产品文档"}'
```

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 知识库名称 |
| description | string | 否 | 知识库描述 |

### 响应

**成功 (200)**：

```json
{
  "id": "kb-uuid",
  "name": "产品手册",
  "description": "公司产品文档",
  "ownerUserId": "user-uuid",
  "createdAt": "2024-01-15T08:30:00Z"
}
```

### 权限

`KB_WRITE`

---

## GET /api/knowledge-bases

查询知识库列表。

### 请求

```bash
curl "http://localhost:8080/api/knowledge-bases?page=0&size=20" \
  -H "satoken: xxx"
```

**查询参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 0 | 页码 |
| size | int | 20 | 每页数量，最大 100 |

### 响应

**成功 (200)**：知识库列表数组。

```json
[
  {
    "id": "kb-uuid",
    "name": "产品手册",
    "description": "公司产品文档",
    "ownerUserId": "user-uuid",
    "createdAt": "2024-01-15T08:30:00Z"
  }
]
```

### 权限

`KB_READ`

### 说明

- `ADMIN` 角色返回所有知识库
- `USER` 角色只返回自己创建的知识库

---

## DELETE /api/knowledge-bases/{id}

删除知识库及其所有文档。

### 请求

```bash
curl -X DELETE http://localhost:8080/api/knowledge-bases/kb-uuid \
  -H "satoken: xxx"
```

### 响应

**成功 (200)**：无响应体。

### 权限

`KB_DELETE`

### 说明

删除知识库会级联删除：
- 所有文档记录
- Milvus 向量数据
- MongoDB 关键词索引
- GridFS 原始文件
- 摄入任务
