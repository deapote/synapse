# API 参考 —— 流式问答

## POST /api/knowledge-bases/{kbId}/query/stream

流式问答接口，使用 SSE（Server-Sent Events）协议返回结果。

### 请求

```bash
curl -N -X POST http://localhost:8082/api/knowledge-bases/kb-uuid/query/stream \
  -H "Content-Type: application/json" \
  -H "synapse-token: xxx" \
  -d '{
    "query": "2024年规定是什么？",
    "sessionId": "session-uuid",
    "asOfDate": "2024-06-01"
  }'
```

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query | string | 是 | 用户问题 |
| sessionId | string | 否 | 会话 ID，用于多轮对话。不传则自动创建新会话 |
| asOfDate | string | 否 | 查询日期，格式 `YYYY-MM-DD`。未传时默认当前日期 |

### asOfDate 查询示例

**查询 2024 年 6 月的适用法规**：

```bash
curl -N -X POST "http://localhost:8082/api/knowledge-bases/{kbId}/query/stream" \
  -H "Content-Type: application/json" \
  -H "synapse-token: xxx" \
  -d '{
    "query": "这项法规的主要内容是什么？",
    "asOfDate": "2024-06-01"
  }'
```

**查询当前适用的法规（不传 asOfDate）**：

```bash
curl -N -X POST "http://localhost:8082/api/knowledge-bases/{kbId}/query/stream" \
  -H "Content-Type: application/json" \
  -H "synapse-token: xxx" \
  -d '{"query": "现行规定是什么？"}'
```

**利用时间意图解析（自动识别）**：

```bash
curl -N -X POST "http://localhost:8082/api/knowledge-bases/{kbId}/query/stream" \
  -H "Content-Type: application/json" \
  -H "synapse-token: xxx" \
  -d '{"query": "2024年新规有什么变化？"}'
```

系统会自动从问题文本中解析出 `2024-12-31` 作为 `asOfDate`。

### 响应

`Content-Type: text/event-stream`

SSE 事件按以下顺序发送：

#### 1. session 事件

```
event: session
data: {"sessionId": "session-uuid"}
```

说明：返回会话 ID。如果是新会话，会创建并返回；如果是已有会话，返回原 ID。

#### 2. token 事件（多次）

```
event: token
data: {"token": "根据"}

event: token
data: {"token": "2024年"}

event: token
data: {"token": "法规"}
```

说明：每个 token 事件包含一个文本片段，前端按顺序拼接即为完整回答。

#### 3. references 事件

```
event: references
data: {"references": [{"sourceId": 1, "documentId": "doc-uuid", "documentName": "法规A-2024版.pdf", "chunkText": "第一条...", "score": 0.9234, "startPosition": 0, "endPosition": 200, "used": true, "canonicalKey": "regulation-a", "versionLabel": "2024 版", "effectiveFrom": "2024-01-01", "effectiveTo": null, "lifecycleStatus": "ACTIVE", "authorityLevel": 10, "jurisdiction": "全国"}]}
```

说明：返回检索到的引用来源列表。每个引用包含完整的时效元数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| sourceId | int | 来源编号，对应回答中的 [1]、[2] |
| documentId | string | 文档 ID |
| documentName | string | 文档名称 |
| chunkText | string | 片段文本 |
| score | float | 相似度分数 |
| startPosition | int | 起始位置 |
| endPosition | int | 结束位置 |
| used | boolean | 回答中是否引用了该来源 |
| canonicalKey | string | 规范标识 |
| versionLabel | string | 版本标签 |
| effectiveFrom | string | 生效日期 |
| effectiveTo | string | 排他结束日期 |
| lifecycleStatus | string | `ACTIVE`/`SUPERSEDED`/`RETIRED` |
| authorityLevel | int | 权威等级 |
| jurisdiction | string | 适用区域 |

#### 4. validation 事件

```
event: validation
data: {"trusted": true, "usedSourceIds": [1], "warnings": []}
```

说明：引用校验结果。

- `trusted`: 是否可信（无警告为 true）
- `usedSourceIds`: 回答中实际引用的来源 ID 列表
- `warnings`: 警告信息列表（如"回答没有引用任何检索来源"）

#### 5. complete 事件

```
event: complete
data: {"sessionId": "session-uuid"}
```

说明：流式输出完成，聊天记录已保存。

#### 6. error 事件（可选）

```
event: error
data: {"message": "流式问答失败", "traceId": "trace-uuid"}
```

说明：处理过程中发生错误。

### 权限

`KB_READ`

### 取消生成

客户端断开连接（如关闭页面或调用 `AbortController.abort()`）会自动取消 LLM 生成。
