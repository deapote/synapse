# 问答检索 —— Adapter-Out 层

## OllamaQueryRewriteAdapter

文件：`synapse-kb-adapter/.../out/llm/OllamaQueryRewriteAdapter.java`

```java
@Component
public class OllamaQueryRewriteAdapter implements QueryRewritePort {
    private final OllamaChatModel chatModel;

    @Override
    public String rewrite(String query) {
        String prompt = """
            请在不改变语义的前提下，将下面的问题改写为更适合知识库检索的一句话。
            只输出改写后的问题，不要解释。

            原问题：%s
            """.formatted(query);

        String rewritten = chatModel.chat(prompt);
        return rewritten.trim();
    }
}
```

Query 改写用同一个 LLM（qwen2.5:7b），但用不同的 prompt。改写后的问题通常更精确、更完整。

## OllamaStreamingLlmAdapter

文件：`synapse-kb-adapter/.../out/llm/OllamaStreamingLlmAdapter.java`

```java
@Component
public class OllamaStreamingLlmAdapter implements StreamingLlmPort {
    private final OllamaStreamingChatModel streamingChatModel;

    @Override
    public Stream<String> generateStream(String prompt) {
        LinkedBlockingQueue<String> queue = new LinkedBlockingQueue<>();
        AtomicBoolean cancelled = new AtomicBoolean(false);

        Thread.startVirtualThread(() -> {
            streamingChatModel.generate(prompt, new StreamingResponseHandler<AiMessage>() {
                @Override
                public void onNext(String token) {
                    if (!cancelled.get()) {
                        queue.offer(token);
                    }
                }

                @Override
                public void onComplete(Response<AiMessage> response) {
                    queue.offer("__COMPLETE__");
                }

                @Override
                public void onError(Throwable error) {
                    queue.offer("__ERROR__" + error.getMessage());
                }
            });
        });

        return StreamSupport.stream(new TokenSpliterator(queue), false)
            .onClose(() -> {
                cancelled.set(true);
                queue.clear();
            });
    }
}
```

### LangChain4j 回调 → JDK Stream 桥接

LangChain4j 的流式 API 是回调式的（`onNext`、`onComplete`、`onError`），但 Application 层需要 JDK 的 `Stream<String>`。这个适配器做了桥接：

1. 创建 `LinkedBlockingQueue` 作为缓冲
2. 用虚拟线程异步消费 LangChain4j 的回调
3. 把 token 放入队列
4. 用 `StreamSpliterator` 从队列中读取，包装成 JDK Stream
5. `Stream.onClose()` 中设置取消标志，停止 LLM 生成

### buildSearchFilter() —— 时效硬过滤

```java
private String buildSearchFilter(KnowledgeBaseId knowledgeBaseId, RetrievalFilter filter) {
    long asOfEpochDay = filter.asOfDate().toEpochDay();
    StringBuilder sb = new StringBuilder();
    sb.append("knowledgeBaseId == '").append(safeFilterValue(knowledgeBaseId.value())).append("'");
    sb.append(" && effectiveFromEpochDay <= ").append(asOfEpochDay);
    sb.append(" && (effectiveToEpochDay == ").append(Long.MAX_VALUE)
      .append(" || effectiveToEpochDay > ").append(asOfEpochDay).append(")");
    sb.append(" && lifecycleStatus in ['ACTIVE', 'SUPERSEDED']");
    if (filter.sourceType() != null) {
        sb.append(" && sourceType == '").append(safeFilterValue(filter.sourceType().name())).append("'");
    }
    if (filter.jurisdiction() != null && !filter.jurisdiction().isBlank()) {
        sb.append(" && jurisdiction == '").append(safeFilterValue(filter.jurisdiction())).append("'");
    }
    return sb.toString();
}
```

**过滤条件解析**：
- `effectiveFromEpochDay <= asOfEpochDay`：资料已经开始生效
- `effectiveToEpochDay == Long.MAX_VALUE || effectiveToEpochDay > asOfEpochDay`：资料尚未过期（`effectiveTo` 是排他结束日）
- `lifecycleStatus in ['ACTIVE', 'SUPERSEDED']`：排除 `RETIRED`
- `sourceType` / `jurisdiction`：可选扩展过滤（v1 保留字段）

## MongoChunkSearchIndexAdapter —— 关键词检索（BM25 时效过滤）

文件：`synapse-kb-adapter/.../out/persistence/MongoChunkSearchIndexAdapter.java`

### search() —— 带时效过滤的 Criteria

```java
@Override
public List<ChunkReference> search(KnowledgeBaseId knowledgeBaseId, String query,
                                    int topK, RetrievalFilter filter) {
    List<String> queryTokens = tokenize(query);
    if (queryTokens.isEmpty()) return List.of();

    List<String> uniqueQueryTokens = new ArrayList<>(new LinkedHashSet<>(queryTokens));
    long asOfEpochDay = filter.asOfDate().toEpochDay();

    Criteria criteria = Criteria.where("knowledgeBaseId").is(knowledgeBaseId.value())
        .and("token").in(uniqueQueryTokens)
        .and("lifecycleStatus").in("ACTIVE", "SUPERSEDED")
        .and("effectiveFromEpochDay").lte(asOfEpochDay)
        .andOperator(
            new Criteria().orOperator(
                Criteria.where("effectiveToEpochDay").is(Long.MAX_VALUE),
                Criteria.where("effectiveToEpochDay").gt(asOfEpochDay)
            )
        );

    if (filter.sourceType() != null) {
        criteria.and("sourceType").is(filter.sourceType().name());
    }
    if (filter.jurisdiction() != null && !filter.jurisdiction().isBlank()) {
        criteria.and("jurisdiction").is(filter.jurisdiction());
    }

    List<ChunkPostingDocument> postings = mongoTemplate.find(
        new Query(criteria).limit(maxCandidates),
        ChunkPostingDocument.class
    );
    // ... BM25 评分和归一化
}
```

### sourceType / jurisdiction 扩展

Milvus 和 Mongo 的检索过滤都支持 `sourceType` 和 `jurisdiction` 作为扩展字段。v1 中这些字段已写入索引，可在后续版本中开放给用户作为检索过滤条件。
