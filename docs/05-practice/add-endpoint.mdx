# 动手实践 —— 给知识库添加标签字段

本节带你完整走一遍"新增一个业务字段"的全流程。假设需求：给知识库添加 `tags` 字段，支持多个标签字符串。

## 需要修改的文件清单

```
Domain        → KnowledgeBase.java（添加 tags 字段和校验）
              → KnowledgeBaseRepository.java（接口不变，标签是内存中的值对象）
Application   → CreateKnowledgeBaseCommand.java（添加 tags 参数）
              → KnowledgeBaseApplicationService.java（传递 tags）
Adapter-In    → KnowledgeBaseController.java（DTO 转 Command）
              → CreateKnowledgeBaseRequest.java（添加 tags 字段）
Adapter-Out   → KnowledgeBaseDocument.java（MongoDB 实体添加 tags）
              → MongoKnowledgeBaseRepository.java（toDocument / toEntity 转换）
Config        → 如果 KnowledgeBaseBeanConfig 手动装配了 Command，可能需要更新
```

## 逐步修改

### 1. Domain 层 —— KnowledgeBase

```java
// synapse-kb-domain/.../model/KnowledgeBase.java

public class KnowledgeBase {
    private final KnowledgeBaseId id;
    private String name;
    private String description;
    private List<String> tags;        // ← 新增
    private final String ownerUserId;
    private final Instant createdAt;

    public static KnowledgeBase create(String name, String description,
                                        List<String> tags, String ownerUserId) {
        // ... 原有校验 ...
        List<String> safeTags = tags == null ? List.of() : List.copyOf(tags);
        return new KnowledgeBase(..., safeTags, ...);
    }

    public static KnowledgeBase reconstruct(KnowledgeBaseId id, String name,
                                             String description, List<String> tags,
                                             String ownerUserId, Instant createdAt) {
        KnowledgeBase kb = new KnowledgeBase(id, name, description,
                tags == null ? List.of() : List.copyOf(tags),
                ownerUserId, createdAt);
        return kb;
    }

    public List<String> getTags() {
        return List.copyOf(tags);  // 防御性拷贝，保持不可变
    }
}
```

**注意**：`List.copyOf()` 保证返回不可变列表，调用方无法修改内部状态。

### 2. Application 层 —— Command + Service

```java
// synapse-kb-application/.../port/in/CreateKnowledgeBaseCommand.java

public record CreateKnowledgeBaseCommand(
    String name,
    String description,
    List<String> tags   // ← 新增
) {}
```

```java
// KnowledgeBaseApplicationService.java:create()

public KnowledgeBaseId create(CreateKnowledgeBaseCommand command) {
    accessControlPort.checkPermission("KB_WRITE");
    KnowledgeBase kb = KnowledgeBase.create(
        command.name(),
        command.description(),
        command.tags(),      // ← 新增
        accessControlPort.currentUserId()
    );
    return knowledgeBaseRepository.save(kb).getId();
}
```

### 3. Adapter-In —— Controller + DTO

```java
// synapse-kb-adapter/.../in/web/KnowledgeBaseController.java

@PostMapping("/api/knowledge-bases")
public Mono<KnowledgeBaseId> create(@RequestBody CreateKnowledgeBaseRequest request) {
    return SaTokenReactorBridge.blockingCall(() ->
        knowledgeBaseApplicationService.create(
            new CreateKnowledgeBaseCommand(
                request.name(),
                request.description(),
                request.tags()   // ← 新增
            )
        )
    );
}
```

```java
// CreateKnowledgeBaseRequest.java

public record CreateKnowledgeBaseRequest(
    String name,
    String description,
    List<String> tags   // ← 新增
) {}
```

### 4. Adapter-Out —— MongoDB 实体和转换

```java
// KnowledgeBaseDocument.java（MongoDB 实体）

@Document(collection = "knowledge_base")
public class KnowledgeBaseDocument {
    // ... 原有字段 ...
    private List<String> tags;   // ← 新增
}
```

```java
// MongoKnowledgeBaseRepository.java

private KnowledgeBaseDocument toDocument(KnowledgeBase kb) {
    KnowledgeBaseDocument doc = new KnowledgeBaseDocument();
    // ... 原有字段 ...
    doc.setTags(kb.getTags());   // ← 新增
    return doc;
}

private KnowledgeBase toEntity(KnowledgeBaseDocument doc) {
    return KnowledgeBase.reconstruct(
        new KnowledgeBaseId(doc.getId()),
        doc.getName(),
        doc.getDescription(),
        doc.getTags(),   // ← 新增
        doc.getOwnerUserId(),
        doc.getCreatedAt()
    );
}
```

### 5. Config 层

如果 `KnowledgeBaseBeanConfig` 手动构造了 `CreateKnowledgeBaseCommand`（通常不会），需要更新。但 Controller 直接 `new` Command，所以 Config 层通常不需要改。
