# 聊天记忆 —— 功能概述

## 场景

用户连续提问：
- 第一轮："这个产品的主要功能是什么？"
- 第二轮："它的价格是多少？"
- 第三轮："和竞品相比有什么优势？"

系统需要记住前面的对话，才能理解"它"指代什么。

## 聊天记忆的设计背景

LLM 是**无状态**的：每次请求都是独立的，它不会自动记住之前的对话。如果我们不主动把历史传入 prompt，LLM 就会"失忆"。

但 LLM 有 **context window（上下文窗口）** 限制：qwen2.5:7b 大约支持 32K token。如果历史太长，就会：
- 超出限制，请求失败
- 挤占检索内容的空间
- 引入噪声，降低回答质量

Synapse 的解决方案：**摘要 + 最近消息**。
