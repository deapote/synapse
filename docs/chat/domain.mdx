# 聊天记忆 —— Domain 层

## ChatSession —— 聊天会话

文件：`synapse-kb-domain/.../model/ChatSession.java`

```java
public class ChatSession {
    private final ChatSessionId id;
    private final String ownerUserId;
    private final KnowledgeBaseId knowledgeBaseId;
    private String title;                    // 会话标题
    private String summary;                  // 历史摘要
    private long summarizedUntilSequence;    // 已摘要到的消息序号
    private long messageCount;               // 消息总数

    public static ChatSession create(String ownerUserId, KnowledgeBaseId knowledgeBaseId) {
        return new ChatSession(
            ChatSessionId.generate(), ownerUserId, knowledgeBaseId,
            "新对话", null, 0, 0
        );
    }

    public long nextSequence() {
        return ++messageCount;
    }

    public void renameFromUserQuestion(String question) {
        if ("新对话".equals(this.title) && question != null) {
            this.title = question.length() > 20
                ? question.substring(0, 20) + "..."
                : question;
        }
    }

    public void updateSummary(String summary, long summarizedUntilSequence) {
        this.summary = summary;
        this.summarizedUntilSequence = summarizedUntilSequence;
    }
}
```

### 设计要点

- `messageCount` 由 `nextSequence()` 原子递增，保证每条消息有唯一序号
- 首次提问时自动重命名标题
- `summarizedUntilSequence` 标记哪些消息已被摘要

## ChatMessage —— 聊天消息

文件：`synapse-kb-domain/.../model/ChatMessage.java`

```java
public record ChatMessage(
    String id,
    ChatSessionId sessionId,
    String ownerUserId,
    KnowledgeBaseId knowledgeBaseId,
    ChatRole role,           // USER 或 ASSISTANT
    String content,
    List<ChunkReference> references,
    long sequence,           // 在会话中的序号
    Instant createdAt
) {}
```

用 `record` 的原因：消息一旦创建就不应该修改，不可变性天然适合消息记录。
