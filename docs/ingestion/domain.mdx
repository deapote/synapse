# 文档上传与摄入 —— Domain 层

## Document —— 文档聚合根

文件：`synapse-kb-domain/.../model/Document.java`

```java
public class Document {
    private final DocumentId id;
    private final KnowledgeBaseId knowledgeBaseId;
    private String fileName;
    private String fileType;
    private long fileSize;
    private final Instant uploadedAt;
    private DocumentStatus status;
    private String failureReason;
    private int chunkCount;
    private String contentHash;
    private String contentObjectId;
    private Instant processingStartAt;
    private Instant processingCompleteAt;

    // 状态机规则表
    private static final Map<DocumentStatus, Set<DocumentStatus>> VALID_TRANSITIONS = Map.of(
        DocumentStatus.PENDING,     EnumSet.of(DocumentStatus.PROCESSING),
        DocumentStatus.PROCESSING,  EnumSet.of(DocumentStatus.COMPLETED, DocumentStatus.FAILED),
        DocumentStatus.FAILED,      EnumSet.of(DocumentStatus.PENDING),
        DocumentStatus.COMPLETED,   EnumSet.noneOf(DocumentStatus.class)
    );
}
```

### 状态机规则

| 当前状态 | 允许转换到 | 说明 |
|---------|-----------|------|
| PENDING | PROCESSING | 开始处理 |
| PROCESSING | COMPLETED | 处理成功 |
| PROCESSING | FAILED | 处理失败 |
| FAILED | PENDING | 重试 |
| COMPLETED | （无） | 终态 |

### create() 工厂方法

```java
public static Document create(KnowledgeBaseId knowledgeBaseId, String fileName,
                               String fileType, long fileSize, String contentHash) {
    if (fileName == null || fileName.isBlank()) {
        throw new DomainException("文件名不能为空");
    }
    return new Document(DocumentId.generate(), knowledgeBaseId, fileName,
                        fileType, fileSize, contentHash, Instant.now());
}
```

**强制 PENDING**：新创建的文档一定是 PENDING 状态，不存在其他可能。

### transitionTo() —— 状态转换

```java
public void transitionTo(DocumentStatus newStatus, String failureReason) {
    Set<DocumentStatus> validNext = VALID_TRANSITIONS.get(this.status);
    if (validNext == null || !validNext.contains(newStatus)) {
        throw new DomainException("无效的状态转换 " + this.status + "->" + newStatus);
    }
    this.status = newStatus;

    // 副作用：自动记录处理时间
    if (newStatus == DocumentStatus.PROCESSING) {
        this.processingStartAt = Instant.now();
        this.processingCompleteAt = null;
        this.failureReason = null;
    }
    if (newStatus == DocumentStatus.COMPLETED || newStatus == DocumentStatus.FAILED) {
        this.processingCompleteAt = Instant.now();
    }
    if (newStatus == DocumentStatus.FAILED && failureReason != null) {
        this.failureReason = failureReason;
    }
}
```

**三步走：查表 → 校验 → 执行 + 副作用。**

### retry() —— 重试

```java
public void retry() {
    transitionTo(DocumentStatus.PENDING);
    this.processingStartAt = null;
    this.processingCompleteAt = null;
    this.failureReason = null;
    this.chunkCount = 0;
}
```

重试时清空所有处理痕迹，相当于"恢复出厂设置"。

## IngestionJob —— 摄入任务

    private final Instant createdAt;
    private Instant updatedAt;
}
```

IngestionJob 有自己的状态机：

```
QUEUED → RUNNING → SUCCEEDED
              ↓
           RETRYING → QUEUED
              ↓
           FAILED
```

### 租约机制

```java
public void claim(String workerId, Duration leaseDuration) {
    this.leaseOwner = workerId;
    this.leaseExpiresAt = Instant.now().plus(leaseDuration);
    this.status = IngestionJobStatus.RUNNING;
}

public boolean isLeaseExpired() {
    return leaseExpiresAt != null && Instant.now().isAfter(leaseExpiresAt);
}
```

**租约机制的作用**

假设 Worker A 认领了任务，开始处理。处理过程中 Worker A 崩溃了，任务一直卡在 RUNNING 状态。其他 Worker 看到 RUNNING 状态就不会处理它，任务永远完不成。

租约机制：Worker A 认领时设置 `leaseExpiresAt = now() + 5分钟`。如果 5 分钟后 Worker A 还没完成，租约过期，其他 Worker 可以重新认领。

## RecursiveChunkingStrategy —— 文本分块

文件：`synapse-kb-domain/.../service/RecursiveChunkingStrategy.java`

```java
public class RecursiveChunkingStrategy {
    private final int maxSegmentSize;   // 最大分块大小（默认 1000 字符）
    private final int overlapSize;      // 重叠窗口大小

    public List<DocumentChunk> split(String text) {
        List<DocumentChunk> chunks = new ArrayList<>();
        int position = 0;

        while (position < text.length()) {
            int end = Math.min(position + maxSegmentSize, text.length());
            int breakPoint = findBreakPoint(text, position, end);

            String chunkText = text.substring(position, breakPoint).strip();
            if (!chunkText.isEmpty()) {
                chunks.add(new DocumentChunk(index, chunkText, position, breakPoint));
            }
            position = findOverlapStart(text, position, breakPoint);
        }
        return chunks;
    }
}
```

### 分块边界优先级

`findBreakPoint()` 按优先级寻找最佳切分点：

1. **Markdown 边界**（标题 `#`、列表 `-`、代码块 ` ``` `）
2. **段落边界**（`\n\n`）
3. **句子边界**（句号、问号、感叹号）
4. **软边界**（空格、标点）

### 分块重叠的设计原因

```
[段落1的后半部分] [段落2的前半部分]
        ↑ 重叠区域 ↑
```

如果句子跨越两个块的边界，没有重叠的话这个句子会被切断，语义丢失。重叠窗口（默认 15%）保证每个块的开头包含上一个块结尾的部分内容，减少信息损失。
