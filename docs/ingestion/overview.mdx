# 文档上传与摄入 —— 功能概述

## 场景

用户在知识库详情页点击"上传文档"，选择一个 PDF 文件。系统需要在后台把这个 PDF 变成可检索的知识：解析文本、切分成小块、向量化、存入向量数据库和关键词索引。

## 文档摄入的复杂度

文档摄入涉及**异步处理**：上传是同步的，但处理是后台进行的。这引入了多个新概念：

- **状态机**：文档有 PENDING → PROCESSING → COMPLETED/FAILED 的状态流转
- **后台 Worker**：定时轮询任务队列
- **租约机制**：防止多个 Worker 同时处理同一个任务
- **补偿逻辑**：每一步失败都要回滚之前的操作
- **双写索引**：向量索引 + 关键词索引必须同时成功
