# Config 层详解

Config 层负责将 Domain 和 Application 层的纯 Java 对象与 Adapter 层的技术实现组装在一起，形成可运行的 Spring Bean。

## 模块结构

```
synapse-kb-config/
└── src/main/java/com/synapse/kb/config/
    └── KnowledgeBaseBeanConfig.java

synapse-auth-config/
└── src/main/java/com/synapse/auth/config/
    ├── AuthBeanConfig.java
    ├── SaTokenSecurityConfig.java
    └── AuthDataInitializer.java
```

## KnowledgeBaseBeanConfig

```java
@Configuration
@ComponentScan("com.synapse.kb.adapter")
@EnableMongoRepositories("com.synapse.kb.adapter.out.persistence")
public class KnowledgeBaseBeanConfig {

    @Bean
    public KnowledgeBaseApplicationService knowledgeBaseApplicationService(
            KnowledgeBaseRepository knowledgeBaseRepository,
            DocumentRepository documentRepository,
            DocumentParserPort documentParserPort,
            EmbeddingPort embeddingPort,
            VectorStorePort vectorStorePort,
            ChunkSearchIndexPort chunkSearchIndexPort,
            QueryRewritePort queryRewritePort,
            StreamingLlmPort streamingLlmPort,
            RecursiveChunkingStrategy chunkingStrategy,
            AccessControlPort accessControlPort,
            ChatSessionRepository chatSessionRepository,
            ChatMessageRepository chatMessageRepository,
            @Value("${rag.top-k:5}") int topK,
            @Value("${rag.prompt-template}") String promptTemplate) {
        return new KnowledgeBaseApplicationService(
            knowledgeBaseRepository, documentRepository,
            documentParserPort, embeddingPort, vectorStorePort,
            chunkSearchIndexPort, queryRewritePort, streamingLlmPort,
            chunkingStrategy, accessControlPort,
            chatSessionRepository, chatMessageRepository,
            topK, promptTemplate
        );
    }

    @Bean
    public RecursiveChunkingStrategy recursiveChunkingStrategy(
            @Value("${chunking.size:1000}") int chunkSize,
            @Value("${chunking.overlap:150}") int chunkOverlap) {
        return new RecursiveChunkingStrategy(chunkSize, chunkOverlap);
    }
}
```

<Info>
  Config 层只负责 Bean 组装，不写业务流程。所有的业务逻辑都在 Application 和 Domain 层。
</Info>

## AuthBeanConfig

```java
@Configuration
public class AuthBeanConfig {

    @Bean
    public AuthApplicationService authApplicationService(
            UserAccountRepository userAccountRepository,
            RoleDefinitionRepository roleDefinitionRepository,
            PasswordHasherPort passwordHasherPort,
            LoginSessionPort loginSessionPort) {
        return new AuthApplicationService(
            userAccountRepository, roleDefinitionRepository,
            passwordHasherPort, loginSessionPort
        );
    }

    @Bean
    public PasswordHasherPort passwordHasherPort() {
        return new BCryptPasswordHasherAdapter();
    }

    @Bean
    public LoginSessionPort loginSessionPort() {
        return new SaTokenLoginSessionAdapter();
    }
}
```

## SaTokenSecurityConfig

```java
@Configuration
public class SaTokenSecurityConfig {

    @Bean
    public SaReactorFilter saReactorFilter() {
        return new SaReactorFilter()
            .addInclude("/api/**")
            .addExclude("/api/auth/login")
            .setAuth(obj -> {
                SaManager.getLog().debug("----- 请求path={}  提交token={}",
                    SaHolder.getRequest().getRequestPath(),
                    StpUtil.getTokenValue());
                StpUtil.checkLogin();
            })
            .setError(e -> {
                return SaResult.error(e.getMessage());
            });
    }
}
```

## 依赖注入设计原则

1. **构造函数注入**：所有 Bean 通过构造函数接收依赖，便于测试
2. **接口依赖**：Application Service 只依赖接口，不依赖具体实现
3. **配置外置**：chunk 大小、topK、prompt 模板等通过 `@Value` 从配置文件读取
4. **无状态 Bean**：Domain Service 和 Application Service 都是无状态的

## Bootstrap 层

```java
@SpringBootApplication
@ComponentScan({
    "com.synapse.kb.config",
    "com.synapse.auth.config"
})
public class SynapseKbBootstrapApplication {
    public static void main(String[] args) {
        SpringApplication.run(SynapseKbBootstrapApplication.class, args);
    }
}
```

Bootstrap 层只负责启动，不直接拼业务对象。所有 Bean 组装都在 Config 层完成。
