# Adapter 层详解

`synapse-kb-adapter` 是知识库上下文的技术实现层，包含入站适配器（Web Controller）和出站适配器（Repository 实现、外部服务调用）。

## 入站适配器（Web Controller）

### DocumentController

```java
@RestController
@RequiredArgsConstructor
public class DocumentController {
    private final IngestDocumentUseCase ingestUseCase;
    private final ListDocumentUseCase listUseCase;
    private final DeleteDocumentUseCase deleteUseCase;

    @PostMapping("/api/knowledge-bases/{kbId}/documents")
    public Mono<DocumentResponse> upload(
            @PathVariable String kbId,
            @RequestPart("file") FilePart file) {
        return DataBufferUtils.join(file.content())
            .map(dataBuffer -> {
                byte[] bytes = new byte[dataBuffer.readableByteCount()];
                dataBuffer.read(bytes);
                DataBufferUtils.release(dataBuffer);
                return bytes;
            })
            .subscribeOn(Schedulers.boundedElastic())
            .map(bytes -> {
                // 校验文件大小、扩展名、MIME
                // 计算 SHA-256 contentHash
                var command = new IngestDocumentCommand(kbId, file.filename(), hash, bytes);
                DocumentId docId = ingestUseCase.ingest(command);
                return new DocumentResponse(docId.value(), ...);
            });
    }
}
```

### StreamingQueryController —— SSE 流式问答

```java
@RestController
@RequiredArgsConstructor
public class StreamingQueryController {
    private final QueryKnowledgeBaseUseCase queryUseCase;
    private final StreamingLlmPort streamingLlmPort;

    @PostMapping(value = "/api/knowledge-bases/{kbId}/query/stream",
                 produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> streamQuery(
            @PathVariable String kbId,
            @RequestBody QueryRequest request) {
        return Mono.fromCallable(() -> {
                Query query = new Query(kbId, request.query());
                return queryUseCase.prepare(query);
            })
            .subscribeOn(Schedulers.boundedElastic())
            .flatMapMany(context -> {
                // session event
                yield ServerSentEvent.<String>builder()
                    .event("session")
                    .data(...)
                    .build();

                // token events
                yield* streamingLlmPort.generateStream(context.prompt())
                    .map(token -> ServerSentEvent.<String>builder()
                        .event("token")
                        .data(token)
                        .build());

                // references event
                yield ServerSentEvent.<String>builder()
                    .event("references")
                    .data(toJson(context.references()))
                    .build();

                // complete event
                yield ServerSentEvent.<String>builder()
                    .event("complete")
                    .data(...)
                    .build();
            });
    }
}
```

## SSE 事件协议

| 事件 | 说明 | 数据 |
|------|------|------|
| `session` | 会话信息 | `{"sessionId":"..."}` |
| `token` | 文本片段（多次） | `{"token":"文本片段"}` |
| `references` | 引用来源 | `{"references":[...]}` |
| `validation` | 引用校验结果 | `{"trusted":true,...}` |
| `complete` | 生成完成 | `{"sessionId":"..."}` |
| `error` | 异常 | `{"message":"...","traceId":"..."}` |

### OllamaEmbeddingAdapter

```java
@Component
@RequiredArgsConstructor
public class OllamaEmbeddingAdapter implements EmbeddingPort {
    private final OllamaEmbeddingModel embeddingModel;

    @Override
    public List<float[]> embed(List<String> texts) {
        return Flux.fromIterable(texts)
            .flatMap(text -> Mono.fromCallable(() ->
                embeddingModel.embed(text).content().vector()
            ).subscribeOn(Schedulers.boundedElastic()))
            .collectList()
            .block();
    }
}
```

使用 `Schedulers.boundedElastic()` 隔离阻塞的 Ollama 网络调用。

### IngestionJobWorker —— 后台摄入任务

```java
@Component
@RequiredArgsConstructor
public class IngestionJobWorker {
    private final ProcessIngestionJobUseCase processUseCase;

    @Scheduled(fixedDelay = 5000)
    public void processPendingDocuments() {
        List<Document> pendingDocs = documentRepository.findByStatus(PENDING);
        for (Document doc : pendingDocs) {
            try {
                processUseCase.process(doc.getId());
            } catch (Exception e) {
                log.error("处理文档失败: {}", doc.getId(), e);
            }
        }
    }
}
```

使用独立 Executor（默认虚拟线程）执行后台任务，避免阻塞 Web 请求处理。
