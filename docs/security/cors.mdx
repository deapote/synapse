# 安全 —— CORS 配置

## 默认配置

CORS 由 `synapse.cors.allowed-origins` 配置控制。默认只允许本地前端：

```yaml
synapse:
  cors:
    allowed-origins: http://localhost:5173,http://127.0.0.1:5173
```

## 生产环境配置

生产环境必须显式配置真实前端域名：

```yaml
synapse:
  cors:
    allowed-origins: https://your-frontend-domain.com
```

## 配置项说明

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `allowed-origins` | `http://localhost:5173` | 允许的前端来源，多个用逗号分隔 |
| `allowed-methods` | GET,POST,PUT,DELETE,OPTIONS | 允许的 HTTP 方法 |
| `allowed-headers` | `*` | 允许的请求头 |
| `allow-credentials` | `true` | 是否允许携带凭证（Cookie/Token） |

## 安全建议

<Danger>
生产环境绝对不要使用 `*` 通配符！这会让任何网站都能调用你的 API。
</Danger>

1. **明确列出所有允许的前端域名**
2. **配合 HTTPS 使用**，防止中间人窃取 Token
3. **定期审查 CORS 配置**，移除不再使用的域名
4. **Token 设置 HttpOnly**（如果使用 Cookie 方案）

## CORS 通配符的安全风险

如果 `allowed-origins: *`：
- 恶意网站 `evil.com` 可以用 AJAX 调用你的 API
- 用户的 `satoken` 会自动携带（因为 `allow-credentials: true`）
- 攻击者可以操作用户的数据

正确的 CORS 配置是**应用安全的第一道防线**。
