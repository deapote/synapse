# 文档上传与摄入 —— Adapter-In 层

## DocumentController

文件：`synapse-kb-adapter/.../in/web/DocumentController.java`

```java
@RestController
public class DocumentController {
    private final IngestDocumentUseCase ingestUseCase;
    private final long maxFileBytes;
    private final List<String> allowedExtensions;

    @PostMapping(value = "/api/knowledge-bases/{kbId}/documents",
                 consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public Mono<DocumentResponse> upload(
            @PathVariable String kbId,
            @RequestPart("file") FilePart filePart) {

        return DataBufferUtils.join(filePart.content())
            .map(dataBuffer -> {
                byte[] bytes = new byte[dataBuffer.readableByteCount()];
                dataBuffer.read(bytes);
                DataBufferUtils.release(dataBuffer);
                return bytes;
            })
            .flatMap(bytes -> SaTokenReactorBridge.blockingCall(() -> {
                // 文件校验
                if (bytes.length > maxFileBytes) {
                    throw new DomainException("文件大小超过限制");
                }
                String fileName = safeFileName(filePart.filename());
                validateFile(fileName);

                // 计算 contentHash
                String contentHash = bytesToHex(
                    MessageDigest.getInstance("SHA-256").digest(bytes)
                );

                // 构建命令并执行
                IngestDocumentCommand command = new IngestDocumentCommand(
                    new KnowledgeBaseId(kbId), fileName,
                    contentType, bytes.length, contentHash,
                    new ByteArrayInputStream(bytes)
                );

                DocumentId documentId = ingestUseCase.ingest(command);
                return new DocumentResponse(documentId.value(), kbId, fileName,
                    contentType, bytes.length, "PENDING", 0, null);
            }));
    }
}
```

### 关键点

**`FilePart`**：WebFlux 的 multipart 文件表示。`filePart.content()` 返回 `Flux<DataBuffer>`，需要 `join()` 合并成完整的字节数组。

**`DataBufferUtils.join()`**：把流式的数据缓冲合并成一个。注意用完后要 `release()`，防止内存泄漏。

**`MessageDigest.getInstance("SHA-256")`**：计算文件的 SHA-256 哈希，用于重复内容检测。

### 文件校验

```java
private void validateFile(String fileName) {
    String ext = getExtension(fileName).toLowerCase();
    if (!allowedExtensions.contains(ext)) {
        throw new DomainException("不支持的文件格式: " + ext);
    }
}
```

校验项：
- 文件大小（默认 10MB）
- 文件扩展名（pdf, doc, docx, txt, md）
- Content-Type

## IngestionJobWorker

文件：`synapse-kb-adapter/.../in/worker/IngestionJobWorker.java`

```java
@Component
@EnableScheduling
public class IngestionJobWorker implements DisposableBean {
    private final ProcessIngestionJobUseCase processUseCase;
    private final ExecutorService executorService;
    private final Semaphore permits;
    private final AtomicBoolean enabled;
    private final String workerId;

    public IngestionJobWorker(ProcessIngestionJobUseCase processUseCase,
                              @Value("${synapse.ingestion.job.enabled:true}") boolean enabled,
                              @Value("${synapse.ingestion.job.concurrency:2}") int concurrency,
                              @Value("${synapse.ingestion.virtual-threads:true}") boolean virtualThreads) {
        this.processUseCase = processUseCase;
        this.executorService = virtualThreads
            ? Executors.newVirtualThreadPerTaskExecutor()
            : Executors.newFixedThreadPool(concurrency);
        this.permits = new Semaphore(concurrency);
        this.enabled = new AtomicBoolean(enabled);
        this.workerId = ManagementFactory.getRuntimeMXBean().getName() + "-" + UUID.randomUUID();
    }

    @Scheduled(fixedDelayString = "${synapse.ingestion.job.poll-delay-ms:1000}")
    public void poll() {
        if (!enabled.get()) return;

        while (permits.tryAcquire()) {
            executorService.execute(() -> {
                try {
                    boolean processed = processUseCase.processNextAvailable(workerId);
                } catch (Exception e) {
                    log.error("摄入任务执行失败", e);
                } finally {
                    permits.release();
                }
            });
        }
    }

    @Override
    public void destroy() {
        executorService.shutdown();
        try {
            if (!executorService.awaitTermination(30, TimeUnit.SECONDS)) {
                executorService.shutdownNow();
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            executorService.shutdownNow();
        }
    }
}
```

### Worker 设计要点

**定时轮询**：`@Scheduled(fixedDelay = 1000)` 每 1 秒执行一次 `poll()`。

**并发控制**：`Semaphore` 限制同时处理的任务数（默认 2）。`tryAcquire()` 非阻塞获取许可，获取不到就不提交新任务。

**虚拟线程**：`Executors.newVirtualThreadPerTaskExecutor()` 创建虚拟线程池。每个任务一个虚拟线程，创建成本极低。

**优雅关闭**：`DisposableBean.destroy()` 确保应用关闭时，等待正在执行的任务完成（最多 30 秒），然后强制关闭。
