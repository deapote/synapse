# 第一次使用

## 1. 登录

打开 `http://localhost:5173`，用默认管理员账号登录：

- 用户名：`admin`
- 密码：`ChangeMe123!`

登录成功后进入知识库列表页面。

## 2. 创建知识库

点击"新建知识库"，输入：

- 名称：`产品手册`
- 描述：`存放公司产品相关文档`

确认后知识库出现在列表中，点击进入详情页。

## 3. 上传文档

在知识库详情页点击"上传文档"，打开上传弹窗。

### 普通文档上传

选择文件（支持 PDF/Word/TXT/Markdown），资料类型保持默认"普通资料"，直接提交即可。和普通网盘上传一样简单。

### 带时效元数据的文档上传

对于法规、政策等需要版本管理的资料，填写以下字段：

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 资料类型 | `LEGAL` / `POLICY` | 法规或政策类型会启用时效治理 |
| 规范标识 | `law-x` | 同一法规多版本的关联键 |
| 版本标签 | `2024 版` | 展示用版本名称 |
| 生效日期 | `2024-01-01` | 资料开始生效的日期 |
| 结束日期 | `2025-01-01` | 排他结束日，到达该日期即失效 |
| 权威等级 | `10` | 冲突时优先级高的资料胜出 |
| 适用区域 | `全国` | 法规适用地域 |
| 替代旧版本 | 下拉选择 | 上传新版本替代已有文档 |

**替代旧版本**的用法：先上传法规 A 的 2024 版，之后上传 2025 修订版时选择"替代旧版本"，旧版会在新文档摄入成功后自动标记为"已替代"。

### 文档状态

上传后文档状态流转：

- **PENDING** — 刚上传，等待处理
- **PROCESSING** — 正在解析、分块、向量化
- **COMPLETED** — 处理完成，可检索
- **FAILED** — 处理失败，可重试

列表中会同时显示：
- **摄入状态**（PENDING/PROCESSING/COMPLETED/FAILED）：文档是否已解析完成
- **时效状态**（ACTIVE/SUPERSEDED/RETIRED）：资料在当前时间点是否有效

二者严格分离，不可混淆。状态刷新周期取决于文档大小和机器性能。

## 4. 问答

文档状态为 COMPLETED 后，点击"开始问答"进入聊天页面。输入与文档内容相关的问题，系统返回基于检索片段生成的答案。

### 适用日期（asOfDate）

聊天输入区上方有"适用日期"控件：

- **不填**：系统默认使用当前日期，检索生效中的资料
- **选择日期**：如选择 `2024-06-01`，系统只检索在该日期有效的资料版本
- **自动识别**：问题中包含"2024年规定"时，后端自动解析为 `2024-12-31`

这对法规类知识库至关重要——问"2024年的规定"时，必须返回 2024 版而非 2025 修订版。

### 引用溯源

回答中的 `[1]`、`[2]` 对应检索到的原始片段。每个引用卡片展示：

- **文档名**和**相似度分数**
- **版本标签**（如"2024 版"）
- **生效期**（如"2024-01-01 ~ 2025-01-01 前有效"）
- **时效状态**（生效中 / 已替代 / 已废止）
- **适用区域**

已替代或已废止的资料会用橙色/红色边框标识，便于快速识别引用资料的时效性。

交互特征：

- **流式输出**：SSE 逐字推送，无需等待全文生成
- **引用溯源**：回答中的 `[1]`、`[2]` 可点击跳转原始片段

## 5. 多轮对话

连续提问时，系统自动携带前文上下文。历史超过阈值后自动压缩为摘要，避免 prompt 超限。

## 6. 多用户隔离（管理员功能）

- 用户名：`testuser`
- 显示名：`测试用户`
- 密码：`Test123456!`
- 角色：`USER`

切换为该用户登录后：

- 只能访问自己创建的知识库
