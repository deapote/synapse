# 认证与权限

## 登录

首次访问系统需要使用管理员账号登录：

| 字段 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `ChangeMe123!` |

<Warning>
  默认管理员密码仅用于开发环境，生产环境请务必修改！
</Warning>

## 权限体系

### 权限列表

| 权限 | 说明 |
|------|------|
| `KB_READ` | 读取知识库、文档列表、问答 |
| `KB_WRITE` | 创建知识库、上传文档 |
| `KB_DELETE` | 删除知识库、删除文档 |
| `AUTH_ADMIN` | 管理用户、角色、权限 |

### 角色定义

| 角色 | 权限 |
|------|------|
| `ADMIN` | 全部权限（KB_READ、KB_WRITE、KB_DELETE、AUTH_ADMIN） |
| `USER` | KB_READ、KB_WRITE |

## 知识库归属

- `USER` 只能访问自己创建的知识库
- `ADMIN` 可跨用户访问所有知识库
- 知识库接口会同时校验权限和知识库归属

## Token 机制

登录成功后，响应包含：

```json
{
  "tokenName": "synapse-token",
  "tokenValue": "eyJ0eXAiOiJKV1QiLCJhbGc..."
}
```

后续所有请求需要在请求头中携带：

```http
synapse-token: eyJ0eXAiOiJKV1QiLCJhbGc...
```
