# API 参考 —— 文档

## POST /api/knowledge-bases/{kbId}/documents

上传文档。

### 请求

```bash
curl -X POST http://localhost:8080/api/knowledge-bases/kb-uuid/documents \
  -H "Content-Type: multipart/form-data" \
  -H "satoken: xxx" \
  -F "file=@/path/to/document.pdf"
```

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| kbId | string | 知识库 ID |

**请求体**：`multipart/form-data`，字段名 `file`。

### 响应

**成功 (200)**：

```json
{
  "id": "doc-uuid",
  "knowledgeBaseId": "kb-uuid",
  "fileName": "document.pdf",
  "fileType": "application/pdf",
  "fileSize": 102400,
  "status": "PENDING",
  "chunkCount": 0,
  "uploadedAt": "2024-01-15T08:30:00Z"
}
```

### 权限

`KB_WRITE`

### 说明

- 上传后文档状态为 `PENDING`，后台自动处理
- 相同内容的文档在同一个知识库中不能重复上传（基于 SHA-256）
- 支持的文件类型：pdf, doc, docx, txt, md

## GET /api/knowledge-bases/{kbId}/documents

查询知识库下的文档列表。

### 请求

```bash
curl "http://localhost:8080/api/knowledge-bases/kb-uuid/documents?page=0&size=20" \
  -H "satoken: xxx"
```

### 响应

**成功 (200)**：文档列表数组。

### 权限

`KB_READ`

---

## DELETE /api/documents/{id}

删除文档。

### 请求

```bash
curl -X DELETE http://localhost:8080/api/documents/doc-uuid \
  -H "satoken: xxx"
```

### 响应

**成功 (200)**：无响应体。

### 权限

`KB_DELETE`

---

## POST /api/documents/{id}/retry

重试失败的文档摄入。

### 请求

```bash
curl -X POST http://localhost:8080/api/documents/doc-uuid/retry \
  -H "satoken: xxx"
```

### 响应

**成功 (200)**：文档信息。

### 权限

`KB_WRITE`

### 说明

- 只有 `FAILED` 状态的文档可以重试
- 重试会清理旧索引并重新处理
