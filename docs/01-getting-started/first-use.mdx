# 第一次使用

现在你已经把项目跑起来了。让我们当一次用户，完整体验所有功能。在这个过程中，你会看到后面要学习的很多技术的"最终效果"。

## 1. 登录

打开 `http://localhost:5173`，用默认管理员账号登录：

- 用户名：`admin`
- 密码：`ChangeMe123!`

登录成功后，你会看到知识库列表页面。目前应该是空的。

**后面你会学到**：登录按钮点击后，前端做了什么？后端怎么验证密码？Session 怎么创建？Token 怎么传递？

→ 见 [切片1：认证登录](/03-vertical-slices/slice-01-auth/story)

## 2. 创建知识库

点击"新建知识库"按钮，输入：

- 名称：`产品手册`
- 描述：`存放公司产品相关文档`

点击确认后，知识库出现在列表中。点击进入详情页。

**后面你会学到**：知识库数据存在哪里？怎么确保 USER 只能看到自己的知识库？ADMIN 怎么能看到所有的？

→ 见 [切片2：文档摄入](/03-vertical-slices/slice-02-document-upload/story)

## 3. 上传文档

在知识库详情页，点击"上传文档"，选择任意一个 PDF/Word/TXT/Markdown 文件（建议选一个你熟悉的文档，方便后面验证问答效果）。

上传后，你会看到文档状态显示为 **PENDING**（待处理）。

过几秒刷新页面，状态变成 **PROCESSING**（处理中）。再等一会儿（取决于文档大小和电脑性能），状态变成 **COMPLETED**（已完成）。

**后面你会学到**：
- 为什么上传后立刻返回，而不是等处理完？
- PENDING → PROCESSING → COMPLETED 的状态机是怎么实现的？
- 后台 Worker 怎么解析 PDF、怎么分块、怎么向量化、怎么存入 Milvus？
- 如果处理失败了怎么办？

→ 见 [切片2：文档摄入](/03-vertical-slices/slice-02-document-upload/story)

## 4. 开始问答

文档状态变成 COMPLETED 后，点击"开始问答"进入聊天页面。

输入一个和文档内容相关的问题，比如如果你的文档是关于产品的，就问"这个产品的主要功能是什么？"

你会看到：

1. **打字机效果**：回答不是一次性出现，而是一个字一个字地显示
2. **引用来源**：回答中可能有 `[1]`、`[2]` 这样的标记，点击可以看到原始文档片段
3. **流式输出**：后端生成一个字，前端立刻显示一个字，不需要等全部生成完

**后面你会学到**：
- 系统怎么找到答案？（向量检索 + 关键词检索 + 融合重排）
- 打字机效果怎么实现的？（SSE 流式输出）
- 引用来源是怎么确定的？
- 系统怎么记住之前的对话？（聊天记忆）

→ 见 [切片3：问答检索](/03-vertical-slices/slice-03-query/story)
→ 见 [切片4：聊天记忆](/03-vertical-slices/slice-04-chat-memory/story)

## 5. 尝试多轮对话

在聊天页面，继续问一个和之前问题相关的问题，比如：

- 第一轮："这个产品的主要功能是什么？"
- 第二轮："它的价格是多少？"
- 第三轮："和竞品相比有什么优势？"

你会发现系统能记住前面的对话上下文。这就是**聊天记忆**功能。

**后面你会学到**：
- 系统不会把全部历史都发给 LLM（会超限），而是只发摘要 + 最近几条消息
- 旧历史达到阈值后会自动压缩成摘要
- 聊天会话和消息是怎么存储的？

→ 见 [切片4：聊天记忆](/03-vertical-slices/slice-04-chat-memory/story)

## 6. 创建新用户（管理员功能）

点击左侧菜单的"用户管理"，创建一个新用户：

- 用户名：`testuser`
- 显示名：`测试用户`
- 密码：`Test123456!`
- 角色：`USER`

然后用这个新用户登录，尝试创建知识库、上传文档、问答。你会发现：

- `testuser` 只能看到自己创建的知识库
- `testuser` 看不到 `admin` 创建的知识库
- `testuser` 不能访问"用户管理"页面（没有 `AUTH_ADMIN` 权限）

**后面你会学到**：
- RBAC 权限模型怎么设计？
- 权限检查在哪里发生？
- 知识库归属怎么校验？

→ 见 [切片1：认证登录](/03-vertical-slices/slice-01-auth/story)

## 验证清单

完成以上步骤后，你应该已经观察到了以下现象：

- [ ] 能正常登录和登出
- [ ] 能创建、查看、删除知识库
- [ ] 能上传文档，看到状态从 PENDING → PROCESSING → COMPLETED
- [ ] 能提问并得到流式回答
- [ ] 能看到引用来源
- [ ] 多轮对话能记住上下文
- [ ] 不同用户只能看到自己的知识库
- [ ] 普通用户不能访问管理功能

如果你都看到了，说明系统运行正常。现在你可以开始阅读后面的章节，学习这些功能是怎么实现的。
