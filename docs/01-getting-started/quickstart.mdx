# 快速开始

本指南帮助你在本地快速搭建 Synapse 开发环境。跟着做，15 分钟后你就能在浏览器里看到一个完整的 RAG 系统。

## 环境要求

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Java | 21+ | 后端运行环境 |
| Maven | 3.9+ | 构建工具 |
| Node.js | 18+ | 前端构建 |
| MongoDB | 6.0+ | 元数据存储 |
| Milvus | 2.3+ | 向量存储 |
| Ollama | 最新 | LLM 和 Embedding 服务 |

## 1. 克隆项目

```bash
git clone https://github.com/sunrise-choir/synapse.git
cd synapse
```

## 2. 启动外部服务

### MongoDB

```bash
docker run -d --name synapse-mongo \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=password \
  mongo:6.0
```

**验证**：
```bash
docker ps | grep synapse-mongo
# 应该看到 synapse-mongo 容器在运行
```

### Milvus

```bash
# 使用 Milvus Standalone 模式
curl -sfL https://raw.githubusercontent.com/milvus-io/milvus/master/scripts/standalone_embed.sh -o standalone_embed.sh
bash standalone_embed.sh start
```

**验证**：
```bash
docker ps | grep milvus
# 应该看到 milvus-standalone 容器在运行
```

### Ollama

```bash
# macOS
brew install ollama
ollama serve

# Linux
curl -fsSL https://ollama.com/install.sh | sh
ollama serve
```

在另一个终端窗口拉取模型：
```bash
# 拉取对话模型（约 4.7GB）
ollama pull qwen2.5:7b

# 拉取 Embedding 模型（约 5GB）
ollama pull gme-qwen2-vl:2b
```

**验证**：
```bash
ollama list
# 应该看到 qwen2.5:7b 和 gme-qwen2-vl:2b
```

## 3. 启动后端

```bash
cd synapse-kb-bootstrap
mvn spring-boot:run
```

**你应该看到的输出**：
```
[INFO] --- spring-boot:3.x.x:run (default-cli) @ synapse-kb-bootstrap ---
...
Tomcat started on port(s): 8082
Started SynapseKbBootstrapApplication in x.x seconds
```

后端默认运行在 `http://localhost:8082`。

## 4. 启动前端

在另一个终端窗口：

```bash
cd synapse-frontend
npm install
npm run dev
```

**你应该看到的输出**：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

前端默认运行在 `http://localhost:5173`。

## 5. 首次登录

打开浏览器访问 `http://localhost:5173`，使用默认管理员账号登录：

| 字段 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `ChangeMe123!` |

<Warning>
  默认管理员密码仅用于开发环境，生产环境请务必修改！
</Warning>

## 6. 验证所有服务正常

登录后，检查以下几点：

- [ ] 页面正常加载，没有报错
- [ ] 能看到"知识库"菜单
- [ ] 点击"新建知识库"能弹出对话框
- [ ] 能成功创建知识库

如果以上都正常，说明后端、前端、MongoDB 都连接成功。接下来你可以跟着 [first-use](/01-getting-started/first-use) 体验完整功能。

## 常见问题

### 端口被占用

```bash
# 检查端口占用
lsof -i :8082  # 后端端口
lsof -i :5173  # 前端端口
lsof -i :27017 # MongoDB 端口
lsof -i :19530 # Milvus 端口
lsof -i :11434 # Ollama 端口
```

如果被占用，可以：
- 杀掉占用进程：`kill -9 <PID>`
- 或修改 `application.yaml` 中的端口配置

### Ollama 模型下载很慢

模型文件很大（共约 10GB），国内网络可能较慢。可以：
- 使用代理加速
- 或先只拉取 qwen2.5:7b，文档摄入和问答需要它；Embedding 模型可以后续再拉

### Maven 依赖下载失败

```bash
# 更换阿里云镜像，编辑 ~/.m2/settings.xml
mvn spring-boot:run -Dmaven.test.skip=true
```

### MongoDB 连接失败

检查 MongoDB 容器是否正常运行：
```bash
docker logs synapse-mongo
```

确认 `application.yaml` 中的连接字符串正确：
```yaml
spring:
  data:
    mongodb:
      uri: mongodb://admin:password@localhost:27017/synapse
```

### 后端启动时卡在"种子数据初始化"

如果看到日志停在 `AuthDataInitializer` 相关输出，可能是 MongoDB 还没准备好。等 MongoDB 完全启动后再重启后端。
