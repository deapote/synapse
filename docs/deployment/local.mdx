# 部署 —— 本地开发环境

## 环境要求

| 组件 | 版本 | 用途 |
|------|------|------|
| Java | 21 | 后端运行 |
| Maven | 3.9+ | 构建 |
| Node.js | 18+ | 前端运行 |
| MongoDB | 6.0+ | 元数据存储 |
| Milvus | 2.3+ | 向量存储 |
| Ollama | 最新 | 本地 LLM |
| MinerU | 3.1.x | 文档解析服务（可回退 Tika） |

## Docker Compose 一键启动依赖

```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  mongodb:
    image: mongo:6
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db

  milvus-standalone:
    image: milvusdb/milvus:v2.3.3
    ports:
      - "19530:19530"
    volumes:
      - milvus_data:/var/lib/milvus

  milvus-attu:
    image: zilliz/attu:v2.3.5
    ports:
      - "8001:3000"
    environment:
      - MILVUS_URL=milvus-standalone:19530

volumes:
  mongo_data:
  milvus_data:
```

```bash
docker-compose -f docker-compose.dev.yml up -d
```

Attu 本地访问端口使用 `8001`，避免和 MinerU API 的 `8000` 冲突。

## 配置文件说明

`synapse-kb-bootstrap/src/main/resources/application.yaml`

```yaml
server:
  port: 8082

spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/synapse_kb?serverSelectionTimeoutMS=2000&connectTimeoutMS=1000

synapse:
  upload:
    max-file-bytes: 20971520
    max-parsed-chars: 10000000
    allowed-extensions: pdf,doc,docx,txt,md
  parser:
    provider: mineru
    mineru:
      base-url: http://127.0.0.1:8000
      endpoint: /file_parse
      backend: pipeline
      parse-method: auto
      language: ch
      formula-enable: true
      table-enable: true
      timeout-seconds: 600
      fallback-to-tika: true
  ingestion:
    virtual-threads: true
  rag:
    top-k: 5
    vector-weight: 0.65
    keyword-weight: 0.35
  chat-memory:
    enabled: true
    recent-message-limit: 8
    summary-trigger-message-count: 12
    max-summary-chars: 1500

ollama:
  base-url: http://localhost:11434
  chat-model: qwen2.5:7b
  embedding-model: hf.co/sinequa/gme-Qwen2-VL-2B-Instruct-GGUF:Q8_0

milvus:
  host: 127.0.0.1
  port: 19530
  embedding-dimension: 1536
  v3:
    collection-name: synapse_document_chunks_v3
```

## 启动 MinerU

本地默认使用独立 MinerU 服务解析 PDF。服务目录与 Synapse 同级：

```text
/Users/admin/codeProject/mineru
```

启动：

```bash
/Users/admin/codeProject/mineru/start-mineru.sh
```

验证：

```bash
curl -sS http://127.0.0.1:8000/health
```

如果暂时不启动 MinerU，可把配置覆盖为 `synapse.parser.provider=tika`，使用内置 Apache Tika 解析。完整 MinerU 部署见 [MinerU 本地解析服务](./mineru-local)。

## 启动后端

```bash
# 方式一：IDE 中运行 SynapseApplication.main()
# 方式二：命令行
mvn clean install -DskipTests
mvn spring-boot:run -pl synapse-kb-bootstrap
```

## 启动前端

```bash
cd synapse-frontend
npm install
npm run dev
```

访问 `http://localhost:5173`
